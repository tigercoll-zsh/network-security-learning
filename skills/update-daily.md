# Update Daily Document Skill (v2.0)

> **重大更新**：本 Skill 已按照"红队导师"教学理念重构，强调"以战促学"。

使用此 Skill 可以快速按照 Day001 的新标准样式更新每日学习文档。

---

## 核心理念

1. **拒绝理论堆砌**：不许一开始就讲 OSI 七层模型
2. **场景驱动**：每天开头必须有"痛点场景"
3. **实战优先**：先动手打靶，再回头补理论
4. **闯关制度**：每天必须有明确的"通关条件"

### 5. 内容生成原则

#### 痛点场景
- 必须是真实发生过的安全事件或常见场景
- 用第二人称"你"来代入
- 引出"为什么要学这个"

#### 实战任务
- **必须放在理论之前！**
- 每个任务都要有明确的操作步骤
- 每个任务都要有预期结果
- 时间估算要合理

#### 底层补课
- 只讲和实验相关的理论
- 用类比和图解，不要干巴巴定义
- 标题用问句更好（如"为什么能抓到明文密码？"）

#### 避坑指南
- 列出初学者最容易卡的地方
- 给出具体的解决方案
- 可以参考社区常见问题

#### 闯关条件
- 必须可验证（如"ping 通"、"成功登录"、"拿到数据"）
- 不能是主观判断（如"理解了"、"掌握了"）

### 6. 清理格式

- 移除 Git 合并冲突标记
- 确保 Markdown 格式正确
- 保持一致的标题层级

### 7. 更新 CHANGELOG

```markdown
### YYYY-MM-DD

#### DayXXX 重构
- **文件**: `daily/DayXXX.md`
- **变更类型**: 按新模板重构
- **变更内容**:
  - 增加痛点场景
  - 调整结构（实战优先）
  - 增加避坑指南
  - 增加闯关条件
```

## 注意事项

1. **保留有价值的内容**：如果目标文档已有技术内容，应保留核心知识点，只调整结构和呈现方式
2. **主题一致性**：生成的痛点场景必须与该天的主题相关
3. **难度递进**：第一阶段要简单直接，第二阶段可以增加复杂度
4. **交叉引用**：在"明日预告"中正确链接后续内容
