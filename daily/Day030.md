## 3. 漏洞详情

### 3.1 SQL 注入漏洞

**风险等级**：高

**漏洞位置**：
- URL: /vulnerabilities/sqli/
- 参数: id

**漏洞描述**：
应用程序未对用户输入进行有效过滤，攻击者可构造恶意 SQL 语句获取数据库信息。

**复现步骤**：
1. 访问 SQL Injection 页面
2. 输入测试 payload: [你的 payload]
3. 观察返回结果

**影响**：
- 可获取数据库全部用户信息
- 可获取管理员密码哈希
- 可能导致进一步的数据库操作

**修复建议**：
1. 使用参数化查询（Prepared Statement）
2. 对用户输入进行严格过滤
3. 最小权限原则配置数据库账户

---

### 3.2 XSS 跨站脚本漏洞

**风险等级**：高

**漏洞位置**：
- URL: /vulnerabilities/xss_s/
- 参数: txtName, mtxMessage

**漏洞描述**：
应用程序未对用户输入进行 HTML 实体编码，攻击者可植入恶意 JavaScript 代码。

**复现步骤**：
1. 访问 XSS (Stored) 页面
2. 在留言板输入: [你的 payload]
3. 提交后刷新页面，观察弹窗

**影响**：
- 可窃取用户 Cookie
- 可劫持用户会话
- 可进行钓鱼攻击

**修复建议**：
1. 对输出内容进行 HTML 实体编码
2. 设置 Cookie HttpOnly 属性
3. 部署 CSP 策略

---

### 3.3 文件上传漏洞

**风险等级**：严重

**漏洞位置**：
- URL: /vulnerabilities/upload/

**漏洞描述**：
文件上传功能仅检查 MIME 类型，可通过修改 Content-Type 绕过上传 PHP 文件。

**复现步骤**：
1. 准备 PHP Webshell 文件
2. 使用 Burp Suite 拦截上传请求
3. 修改 Content-Type 为 image/jpeg
4. 放行请求，成功上传

**影响**：
- 可上传 Webshell
- 可获取服务器命令执行权限
- 可完全控制服务器

**修复建议**：
1. 使用后缀名白名单
2. 对上传文件重命名
3. 上传目录禁止执行权限
4. 检测文件真实内容
