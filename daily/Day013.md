### 任务 2：报告模板（10 分钟）

创建报告文件 `recon_report.md`，使用以下结构：

```markdown
# 渗透测试侦察报告

## 1. 执行摘要
- 目标概述
- 关键发现
- 风险等级

## 2. 目标信息
- 域名/IP
- 网络架构
- 联系信息

## 3. 信息收集
### 3.1 DNS 信息
### 3.2 WHOIS 信息
### 3.3 子域名

## 4. 端口和服务
- 开放端口列表
- 服务版本
- 运行截图

## 5. 漏洞分析
- 潜在漏洞列表
- CVE 编号
- 严重性评估

## 6. 攻击路径
- 推荐的攻击顺序
- 所需工具

## 7. 附录
- 原始扫描结果
- 工具输出
```

```markdown
## 1. 执行摘要

**目标**：Metasploitable2 (192.168.56.102)

**关键发现**：
- 目标运行 20+ 个开放端口
- 多个服务存在已知严重漏洞
- vsftpd 2.3.4 存在后门（CVE-2011-2523）
- Samba 3.0.20 存在远程代码执行漏洞

**风险等级**：严重

**建议**：该系统存在多个可被远程利用的漏洞，建议立即处理。
```

## 底层补课

### 为什么要写报告？

1. **整理思路**：把散乱的信息系统化
2. **专业要求**：真实渗透测试必须有报告
3. **证据留存**：记录你做了什么、发现了什么
4. **团队协作**：让其他人能理解和接手

### 好报告的特点

| 特点 | 说明 |
|------|------|
| 结构清晰 | 按标准模板 |
| 信息完整 | 不遗漏重要发现 |
| 可复现 | 别人能根据报告重做 |
| 有证据 | 截图、日志、命令 |
| 有建议 | 不只是发现问题，还要给方案 |

### 常用报告工具

| 工具 | 用途 |
|------|------|
| Markdown | 简单文本报告 |
| Obsidian | 知识管理 |
| CherryTree | 渗透测试笔记 |
| Dradis | 专业渗透报告 |

## 避坑指南

| 坑点 | 症状 | 解决方案 |
|------|------|----------|
| 信息散乱 | 找不到之前的扫描结果 | 从一开始就分类保存 |
| 缺少证据 | 只有结论没有截图 | 边做边截图 |
| 不可复现 | 忘了用什么命令 | 记录每条命令 |

## 常见问题

**Q1：报告写多长合适？**

A1：质量比数量重要。侦察阶段的报告，3-5 页足够。重点是信息完整、结构清晰。

**Q2：需要包含原始扫描输出吗？**

A2：重要的放正文，完整输出放附录。方便以后查阅。

**Q3：报告用什么格式？**

A3：Markdown 最灵活，可以转 PDF、HTML。企业环境可能要求 Word 或专用工具。
