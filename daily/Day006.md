- 公钥信息

## 📖 底层补课

### HTTPS 解决了什么问题？

| 问题 | HTTP | HTTPS |
|------|------|-------|
| 窃听 | ❌ 明文可偷看 | ✅ 加密 |
| 篡改 | ❌ 可修改内容 | ✅ 完整性校验 |
| 冒充 | ❌ 可假冒网站 | ✅ 证书验证 |

### TLS 握手流程（简化版）

```
客户端                          服务器
   │                              │
   │── Client Hello ─────────────→│  1. "我支持这些加密算法"
   │                              │
   │←── Server Hello + 证书 ──────│  2. "用这个算法，这是我的证书"
   │                              │
   │── 验证证书 + 生成密钥 ───────→│  3. 客户端验证后生成对称密钥
   │                              │
   │←───── 加密通信开始 ──────────→│  4. 后续用对称密钥加密
```

### 对称加密 vs 非对称加密

| 类型 | 速度 | 密钥 | 用途 |
|------|------|------|------|
| 对称加密 | 快 | 一个密钥 | 加密数据 |
| 非对称加密 | 慢 | 公钥+私钥 | 交换对称密钥 |

**HTTPS 两者都用**：用非对称加密安全地交换对称密钥，然后用对称加密传输数据。

## ❓ 常见问题

**Q1：HTTPS 就绝对安全吗？**

A1：不是。如果：
- 你电脑装了恶意根证书
- 你点击了忽略证书警告
- 服务器本身被黑了

那 HTTPS 也保护不了你。

**Q2：为什么还有网站用 HTTP？**

A2：懒、便宜、不知道。Let's Encrypt 提供免费证书后，已经没有理由不用 HTTPS 了。
