> **场景**：你在星巴克用公共 WiFi 登录了一个 HTTP 网站。隔壁桌的黑客打开 Wireshark，把整个咖啡厅的流量全抓了。
>
> **结果**：你的密码被截获了。

## ⚔️ 实战任务（先动手！）

### 任务 1：启动 Wireshark（5 分钟）

```bash
sudo wireshark
```

**选择网卡**：
- 虚拟机内部通信：选择 `lo`（loopback）或 `eth0`
- Host-Only 网络：选择对应的网卡

点击网卡名称开始捕获。

---

### 任务 2：访问 DVWA 并登录（5 分钟）

1. **不要**通过 Burp 代理（直接访问）
2. Firefox 访问 `http://127.0.0.1`
3. 输入 admin / password 登录
4. 回到 Wireshark，点击停止捕获（红色方块）

---

### 任务 3：过滤 HTTP 流量（10 分钟）

在过滤栏输入：
```
http
```

或更精确：
```
http.request.method == "POST"
```

你应该能看到 POST /login.php 的请求。

---

### 任务 4：查看明文密码（10 分钟）

1. 找到 POST 请求，点击选中
2. 下方展开 **Hypertext Transfer Protocol**
3. 展开 **HTML Form URL Encoded**
4. 看到：`username=admin` 和 `password=password`

**或者**：右键 → Follow → HTTP Stream，看到完整对话。

## 📖 底层补课

### Wireshark vs Burp Suite

| 工具 | 抓包层级 | 原理 |
|------|----------|------|
| Burp Suite | 应用层（代理） | 浏览器主动发给它 |
| Wireshark | 网络层（嗅探） | 偷听网卡上的所有流量 |

### 为什么能抓到别人的流量？

在共享网络（如公共 WiFi）下，如果没有隔离措施，所有流量都会经过你的网卡。

### 常用过滤器

| 过滤器 | 作用 |
|--------|------|
| `http` | 只显示 HTTP |
| `ip.addr == 192.168.1.1` | 指定 IP |
| `tcp.port == 80` | 指定端口 |
| `http.request.method == "POST"` | 只看 POST |

## ⚠️ 避坑指南

| 坑点 | 症状 | 解决方案 |
|------|------|----------|
| 抓不到本机流量 | 没有包 | 选择 `lo` 网卡 |
| 权限不足 | 打不开网卡 | 用 `sudo wireshark` |
| 流量太多 | 找不到目标 | 用过滤器 |
