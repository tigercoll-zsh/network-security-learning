## 🔥 今日痛点场景

> **场景**：你在某个 HTTP 网站登录，输入了用户名和密码。你以为只有服务器能看到这些信息。
>
> **真相**：同一个 WiFi 下的任何人，用抓包工具就能看到你的密码。
>
> **今天的任务**：成为那个"能看到密码的人"——用 Burp Suite 拦截登录请求。

---

## 🎯 今日目标

- [ ] 启动 Burp Suite 并配置代理
- [ ] 让浏览器的流量经过 Burp
- [ ] 拦截一个 HTTP 请求并查看内容
- [ ] 在请求中找到用户名和密码

**闯关条件**：在 Burp Suite 中看到 DVWA 登录请求里的明文密码。

---

## ⚔️ 实战任务（先动手！）

### 任务 1：启动 Burp Suite（5 分钟）

```bash
burpsuite
```

选择 Temporary project → Use Burp defaults → Start Burp

---

### 任务 2：配置代理（10 分钟）

**Burp 设置**：确认 Proxy → Proxy settings 中有 `127.0.0.1:8080`

**Firefox 设置**：
1. Settings → 搜索 proxy
2. Manual proxy configuration
3. HTTP Proxy: `127.0.0.1`，Port: `8080`
4. 勾选 Also use this proxy for HTTPS

---

### 任务 3：拦截登录请求（20 分钟）

1. Burp 中开启 **Intercept is on**
2. Firefox 访问 DVWA 登录页
3. 输入 admin / password，点击 Login
4. 切换到 Burp，查看拦截的 POST 请求
5. 找到 `username=admin&password=password`

**这就是明文密码！**

## 📖 底层补课（做完实验再看）

### 为什么能看到明文密码？

HTTP 协议是**明文传输**的，就像寄明信片，路上任何人都能看。

### 什么是代理？

代理就像"中间商"，Burp 站在浏览器和服务器中间，可以查看和修改所有通信。

### HTTP 请求结构

```http
POST /login.php HTTP/1.1
Host: 127.0.0.1
Content-Type: application/x-www-form-urlencoded

username=admin&password=password
```

## ⚠️ 避坑指南

| 坑点 | 症状 | 解决方案 |
|------|------|----------|
| 浏览器一直转圈 | 页面加载不出来 | 去 Burp 点 Forward 放行！ |
| 看不到 POST 请求 | 只有 GET | 确保 Intercept is on |

## 💡 知识卡片速查

### HTTP 状态码

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 302 | 重定向 |
| 403 | 禁止访问 |
| 404 | 找不到 |
| 500 | 服务器错误 |
