### 任务 2：威胁建模（50 分钟）

**STRIDE 威胁模型**：

| 威胁 | 全称 | 说明 | 对策 |
|------|------|------|------|
| **S** | Spoofing | 身份欺骗 | 认证 |
| **T** | Tampering | 篡改 | 完整性校验 |
| **R** | Repudiation | 抵赖 | 日志审计 |
| **I** | Information Disclosure | 信息泄露 | 加密 |
| **D** | Denial of Service | 拒绝服务 | 可用性保障 |
| **E** | Elevation of Privilege | 权限提升 | 最小权限 |

**威胁建模流程**：

```
1. 资产识别 → 2. 威胁识别 → 3. 风险评估 → 4. 对策制定 → 5. 验证
     ↓              ↓              ↓              ↓           ↓
  核心资产      STRIDE      影响 × 可能性     安全控制     渗透测试
```

**实操练习**：为一个 Web 应用做威胁建模

| 资产 | 威胁 | 风险 | 对策 |
|------|------|------|------|
| 用户数据 | 信息泄露 | 高 | 加密存储、访问控制 |
| 登录功能 | 身份欺骗 | 高 | MFA、密码策略 |
| API 接口 | 篡改 | 中 | 签名验证、HTTPS |
| 服务可用 | DoS | 中 | CDN、限流 |

### 任务 3：安全运营中心（SOC）（40 分钟）

**SOC 核心职能**：

| 职能 | 说明 | 工具 |
|------|------|------|
| 监控 | 7×24 安全监控 | SIEM |
| 检测 | 威胁检测分析 | IDS、EDR |
| 响应 | 事件响应处置 | SOAR |
| 狩猎 | 主动威胁狩猎 | Hunting 平台 |
| 情报 | 威胁情报整合 | TIP |

**SOC 技术栈**：

```
┌─────────────────────────────────────────────────────────┐
│                    SOAR (自动化响应)                     │
├─────────────────────────────────────────────────────────┤
│                    SIEM (日志关联分析)                   │
├─────────────────────────────────────────────────────────┤
│       日志源：防火墙 / IDS / EDR / WAF / 系统日志       │
└─────────────────────────────────────────────────────────┘
```

**关键指标**：

| 指标 | 说明 | 目标 |
|------|------|------|
| MTTD | 平均检测时间 | 越短越好 |
| MTTR | 平均响应时间 | 越短越好 |
| 误报率 | 误报占比 | < 10% |
| 覆盖率 | 资产监控覆盖 | > 95% |

### 任务 4：防御策略制定（40 分钟）

**策略制定框架**：

```markdown
# 安全防御策略

## 1. 现状评估
- 资产清单
- 现有安全能力
- 历史安全事件
- 合规要求

## 2. 风险分析
- 威胁来源（外部攻击、内部威胁、供应链）
- 脆弱性评估
- 风险优先级排序

## 3. 防御目标
- 短期目标（0-3 个月）
- 中期目标（3-12 个月）
- 长期目标（1-3 年）

## 4. 技术方案
- 安全架构设计
- 产品选型
- 部署计划

## 5. 运营方案
- 监控策略
- 响应流程
- 人员配置

## 6. 度量指标
- KPI 定义
- 评估周期
- 持续改进
```

**实操练习**：为一个中小企业制定防御策略

**场景**：50 人的互联网公司，有一个 Web 产品

| 阶段 | 优先级 | 措施 |
|------|--------|------|
| 短期 | P0 | 基础加固、备份、WAF |
| 短期 | P1 | 漏洞扫描、补丁管理 |
| 中期 | P0 | SIEM 部署、日志收集 |
| 中期 | P1 | SDL 建设、代码审计 |
| 长期 | P0 | SOC 建设、红蓝对抗 |
| 长期 | P1 | 零信任架构 |

### 零信任原则

1. **永不信任，始终验证**
2. **最小权限访问**
3. **假设被入侵**
4. **微分段隔离**
